## üìù Documentation Standards

### File Naming
- Use lowercase with hyphens: `my-document.md`
- Be descriptive: `payment-flow.md` not `payments.md`
- Group related docs in folders

### Markdown Format
- Start with H1 title
- Include a brief overview/summary
- Use proper heading hierarchy (H1 ‚Üí H2 ‚Üí H3)
- Add code blocks with language identifiers
- Include examples where helpful

### Keeping Docs Updated
- Update docs when code changes significantly
- Mark outdated docs with a note at the top
- Move completed project docs to `archive/`
- Don't duplicate information (link instead)
